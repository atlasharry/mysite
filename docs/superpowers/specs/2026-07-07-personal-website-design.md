## 1. 目标与受众

展示余城宇的双重身份——青年导演 × 计算机科学研究者/工程师——并融入精简履历与联系方式。

受众两者并重：

- 电影节、制片方、影像行业合作者（中文为主）
- 招聘方、学术合作者（英文为主）

气质基准：暗色电影感、画廊式留白、星空元素贯穿；克制、大牌质感，类 Anthropic 的干净排版。明确反面：土气、花里胡哨、堆砌动效。

## 3. 视觉语言

### 3.1 色彩（Anthropic 式暖调莫兰迪，全站无高饱和色）

| 用途 | 色值 |
|------|------|
| 页面背景（暖黑） | `#141413` |
| 章节暗场/加深背景 | `#0E0D0C` |
| 正文（象牙白） | `#FAF9F5` |
| 次要文字（暖灰） | `#B0AEA5` |
| 强调色（奶油黄，唯一点缀色） | `#EBDBBC` |

强调色只用于：章节编号、title card 细线、链接 hover 微光、画廊标注。用量目标 < 5% 画面面积。星点用象牙白/奶油色调。

### 3.2 字体（全部子集化 woff2 自托管，不走 Google Fonts）

- 中文标题：思源宋体 Noto Serif SC（电影字幕卡气质）
- 中文正文：思源黑体 Noto Sans SC
- 英文标题：Lora（Anthropic 正文字体，衬线）
- 英文正文：系统 sans 栈或 Noto Sans 拉丁部分
- 排版：大字号层级、大行距、正文行长 ≤ 65ch、大量留白

### 3.3 章节 title card

每章开场：`01 ──── 影像 FILMS` —— 奶油黄编号 + 细线 + 中英标题，从纯黑中淡入并轻微上浮，模拟片头字幕卡。

### 3.4 星空的贯穿叙事

- 首屏：星空最浓（canvas 星点、缓慢闪烁、滚动视差下沉淡出）
- 中间章节：仅极稀疏星点底纹，若有若无
- 「星空」章节：星空浓密回归，与首屏呼应
- 原则：不在每屏撒星星

## 4. 转场与动效（克制原则：宁少勿多）

1. **章节切换 = 暗场（cut to black）**：章节间纯黑缓冲带，下一章 title card 从黑中淡入。全站统一使用此转场语法。
2. **唯一的特殊转场**：影像 → AI×创作 之间，胶片颗粒噪点随滚动缓慢重组为星点/数据网格（模拟到数字的隐喻）。仅此一处。
3. **元素入场**：IntersectionObserver 触发 fade + 上移约 24px，~700ms 电影缓动（cubic-bezier），只播一次。
4. **画廊**：hover 轻微缩放 + 地点标注浮现；点击进 lightbox（黑场大图、左右切换、ESC 关闭）。
5. **无障碍**：遵守 `prefers-reduced-motion`，减少动态时禁用视差与粒子动画。

## 6. 工程架构

技术方案（已确认）：纯静态手工，无框架无构建。

```
mysite/
├─ index.html          # 单页结构
├─ css/style.css       # 设计系统 + 版式 + 动效
├─ js/
│   ├─ data.js         # 全站中英文案 + 影片数据 + 照片清单（唯一内容源）
│   ├─ i18n.js         # 语言切换
│   ├─ starfield.js    # 星空 canvas
│   └─ transitions.js  # 滚动转场 / IntersectionObserver / lightbox
├─ assets/
│   ├─ films/          # 海报、剧照（已优化）
│   ├─ astro/          # 星空极光精选（已优化）
│   ├─ travel/         # 旅行精选（已优化）
│   ├─ raw/            # 用户投放原图（gitignore，不上线）
│   └─ fonts/          # 子集化 woff2
├─ resume/             # 网页安全版简历 PDF
├─ tools/optimize.py   # 一次性图片压缩脚本（raw → WebP+JPG 双格式、生成缩略图、写入清单）
└─ docs/superpowers/specs/  # 本文档
```

- 内容更新流：改 `js/data.js`（或往 `assets/raw/` 投图后跑 `tools/optimize.py`）→ git push → 自动部署
- 图片规格：大图长边 ≤ 2000px WebP（JPG 兜底），画廊缩略图长边 ~640px；全部懒加载

## 7. 部署

- 仓库：`mysite/` 即 git 仓库，推送到 GitHub
- 主线：Cloudflare Pages 连接 GitHub 仓库，push 即自动部署（免费、全球 CDN、大陆通常可访问、无需备案）
- 备线：GitHub Pages 同仓库开启
- 域名：可选，后续在 Cloudflare 绑定；默认使用 `*.pages.dev`

## 8. 隐私红线

- 网站与仓库中**不出现**：手机号、住址、出生日期、亲属姓名及联系方式
- 简历原版 PDF 不上线；生成去除手机号/住址的网页安全版 PDF 供下载
- 公开联系方式仅限用户指定的邮箱与社交链接
- FIRST 申报表中的制片人/演员个人信息不上线（片名主创署名除外）
- `assets/raw/` 加入 `.gitignore`，原图不进仓库

## 9. 性能预算

- 首屏传输 < 1MB；LCP < 2.5s（普通 4G）
- 图片全懒加载；字体子集化；无第三方请求（无 CDN 脚本、无统计脚本、无外部字体）

## 10. 明确不做（v1 范围外）

- 不嵌入任何视频平台播放器（B 站/YouTube/Vimeo）
- 不做浅色模式（暗色即定位）
- 不做多页面路由、不做 CMS、不做评论/留言
- 不做微信朋友圈内容提取
- 不放《生日礼物》正片或任何公开放映形式的片段
